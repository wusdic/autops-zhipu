## 1. 测试分层

```text
E2E 测试 (Playwright)
  ↑
集成测试 (pytest)
  ↑
单元测试 (pytest)
```

---

## 2. 单元测试

### 覆盖范围

| 领域 | 测试重点 |
|---|---|
| 资产 | CRUD、生命周期、关系、分组 |
| 配置 | 版本化、发布、回滚、漂移检测 |
| 凭证 | 加密解密、绑定、测试 |
| 采集 | normalize、状态变更、错误分类 |
| 事件 | 去重、关联、分类 |
| 告警 | 规则匹配、收敛、抑制、升级 |
| 策略 | 条件匹配、模拟、冲突检测 |
| 自动化 | 状态机、dry-run、高危命令检测 |
| AIops | 输出解析、降级 |
| 权限 | RBAC、数据权限 |

### 目标

核心业务逻辑覆盖率 **80%+**

### 工具

- pytest + pytest-asyncio
- pytest-cov（覆盖率）
- factory_boy（测试数据工厂）
- faker（测试数据生成）

## 3. 集成测试

### 完整链路测试

```text
资产 → 配置 → 采集 → 状态 → 事件 → 告警
→ 策略 → 自动化 → 日志 → 验证 → 工单 → 知识
```

### 覆盖场景

| 场景 | 说明 |
|---|---|
| 采集→状态→事件 | 采集异常→状态变更→事件生成 |
| 事件→告警→策略 | 事件触发告警→策略匹配 |
| 策略→执行→验证 | 策略触发执行→执行完成→验证 |
| 执行失败→回滚 | 执行失败→自动回滚 |
| 执行失败→工单 | 执行失败→自动转工单 |
| AI 分析→推荐→执行 | AI 分析→策略校验→执行 |
| 工单→知识 | 工单关闭→知识草稿生成 |

### 工具

- pytest + httpx（API 测试）
- testcontainers（数据库容器）
- Docker Compose（完整环境）

## 4. E2E 测试

### 覆盖流程

1. 登录
2. 创建资产
3. 绑定凭证
4. 触发采集
5. 查看告警
6. 查看 AI 分析
7. 执行 dry-run
8. 执行自动化
9. 查看实时日志
10. 查看验证结果
11. 生成知识草稿

### 工具

- Playwright

## 5. 安全测试

| 检查项 | 说明 |
|---|---|
| Secret 扫描 | 代码中无明文密钥/密码 |
| 凭证安全 | 数据库中无明文凭证 |
| 动态脱敏 | API 响应中敏感字段脱敏 |
| 权限越权 | 低权限用户不能访问高权限 API |
| 高危阻断 | 高危命令被阻断 |
| API Key 越权 | Key 只能访问 scope 内资源 |
| 审计完整性 | 所有敏感操作有审计记录 |
| Token 安全 | Token 过期、刷新正常 |

## 6. 容错测试

| 场景 | 说明 |
|---|---|
| 数据库异常 | 服务降级但不崩溃 |
| Redis 异常 | 缓存不可用时回退数据库 |
| 模型服务异常 | AI 分析降级，不阻塞平台 |
| 采集器离线 | 生成离线事件 |
| Worker 失败 | 任务重试 |
| WebSocket 断开 | 自动重连 |
| 脚本执行失败 | 状态正确流转 |
