## 2. 必须展示的信息

| 区域 | 信息 |
|---|---|
| **告警头部** | 告警标题、严重度、触发时间、持续时间、状态 |
| **时间线** | 按时间排列的事件节点（告警触发→AI分析→策略匹配→执行→验证→关闭） |
| **指标** | 相关指标趋势图（最近 24h） |
| **日志** | 相关日志片段（最近错误日志） |
| **资产** | 资产信息、关系图、当前状态 |
| **AI 分析** | 摘要、可能原因（含置信度）、推荐动作、不确定性 |
| **策略** | 命中策略名称、命中原因、风险等级、动作链 |
| **操作** | dry-run、执行、审批、取消、回滚、转工单 |
| **日志流** | 实时 stdout/stderr 流（执行时） |
| **验证** | 执行后验证结果 |

## 3. 交互流程

### 3.1 自动处置

```text
告警触发 → 自动 AI 分析 → 自动策略匹配 → 展示推荐
  → 用户点击 [模拟执行] → 查看 dry-run 结果
  → 用户点击 [执行] → 审批（如需）→ 实时日志流
  → 执行完成 → 自动验证 → 展示结果
  → 成功：关闭告警
  → 失败：展示失败原因，可选回滚或转工单
```

### 3.2 手动处置

```text
告警触发 → 查看告警上下文 → 用户判断
  → 手动选择脚本/Playbook → 配置参数 → 执行
  → 或转工单
```
