# AUTOPS 前端设计规范（Design System）

> 版本：3.0 | 三轮迭代最终版  
> 参照：Ant Design Pro、DataDog、Grafana、阿里云控制台、Stripe Dashboard

---

## 一、设计哲学

### 1.1 核心原则
1. **信息层级清晰**：一眼看到最重要的数据，3秒找到入口
2. **操作流连贯**：从发现→诊断→处置→复盘，路径不断裂
3. **留白有度**：不是越紧凑越好，也不是越宽松越好，间距服务于信息分组
4. **一致性**：相同操作相同交互，相同数据相同展示

### 1.2 信息优先级（运维场景）
```
P0 告警/异常（红色，必须立即看到）
P1 关键指标（橙/黄色，需要关注）
P2 常规数据（蓝色/灰色，正常浏览）
P3 辅助信息（浅灰，按需展开）
```

## 二、布局系统

### 2.1 整体布局
```
┌────────────────────────────────────────────────────┐
│ Header（56px固定高度）                              │
├──────────┬─────────────────────────────────────────┤
│          │                                         │
│ Sidebar  │          Main Content                   │
│ (200px   │          (padding: 20px)                │
│  /64px   │                                         │
│  折叠)    │                                         │
│          │                                         │
│          │                                         │
└──────────┴─────────────────────────────────────────┘
```

### 2.2 Sidebar 规范
- **展开宽度**：200px（不是220px，不是256px）
- **折叠宽度**：64px
- **背景色**：`#1d2129`（深炭灰，比纯黑柔和）
- **文字色**：`#c9cdd4`（主），`#86909c`（次），`#ffffff`（激活）
- **激活态**：左侧3px蓝色竖条 + 背景色 `#2a323d`
- **子菜单**：默认折叠，点击展开（不要全部展开！）
- **折叠态**：hover显示tooltip
- **菜单项高度**：44px（不是40px，也不是48px）
- **字号**：14px（菜单项），12px（子标题）
- **图标**：20px，与文字间距8px

### 2.3 Header 规范
- **高度**：56px
- **背景**：`#ffffff`
- **底部边框**：`1px solid #e5e6eb`
- **内容**：左（折叠按钮+面包屑），右（搜索+通知+用户菜单）
- **用户菜单**：点击弹出下拉（个人设置/退出登录），不要只显示名字

### 2.4 Main Content 规范
- **背景**：`#f7f8fa`
- **padding**：20px（统一，不要有的16px有的24px）
- **max-width**：无限制（自适应填满）
- **内容卡片间距**：16px

## 三、卡片系统

### 3.1 统计卡片（Metric Card）
```
┌──────────────────┐
│ 📊 告警总数       │  ← 标题行：图标+标签，14px，#86909c
│                  │
│ 42               │  ← 数值：32px，bold，#1d2129
│ ↑ 12% vs昨日     │  ← 趋势：12px，绿↑/红↓
└──────────────────┘
高度：100px，内padding：16px
hover：阴影提升
```

### 3.2 数据卡片（Data Card）
```
┌──────────────────────────┐
│ 📋 标题          [操作▶] │  ← header：标题左，操作右
├──────────────────────────┤
│                          │
│    内容区域               │  ← body：表格/图表/列表
│                          │
└──────────────────────────┘
header高度：48px，padding：16px
body padding：16px
border-radius：8px（不是4px，不是0）
border：1px solid #e5e6eb
shadow：none（默认），hover时 0 2px 8px rgba(0,0,0,0.06)
```

### 3.3 卡片间距
- 同行卡片间距：16px（gutter）
- 跨行卡片间距：16px
- 卡片组与卡片组之间：20px

## 四、栅格系统

### 4.1 栅格规则
- 一行24份（Element Plus标准）
- 统计卡片：一行4-6个 → span=6或span=4
- 内容+侧边：span=16 + span=8，或 span=18 + span=6
- 全宽表格：span=24
- 双栏表单：span=12 + span=12

### 4.2 响应式断点
```
xs: <768px   → 单列
sm: ≥768px   → 双列
md: ≥992px   → 三列
lg: ≥1200px  → 四列
xl: ≥1920px  → 五/六列
```

---

## 五、间距规范（Spacing Scale）

```
4px  → 微间距（图标与文字）
8px  → 小间距（表单元素之间）
12px → 中间距（卡片内元素分组）
16px → 标准间距（卡片之间、栅格gutter）
20px → 大间距（页面padding、区域之间）
24px → 特殊间距（标题与内容之间）
32px → 极大间距（很少使用）
```

**核心规则**：同一层级用相同间距，不同层级用不同间距。

## 八、表格规范

### 8.1 表格样式
- 斑马纹：开启（`stripe`）
- 边框：横向边框（`border`）
- 行高：默认48px（不要太紧凑）
- 表头：背景 `#fafafa`，文字 `#86909c`，14px，500字重
- 空状态：图标+文字+操作引导

### 8.2 表格操作列
- 固定右侧（`fixed="right"`）
- 按钮用 `text` 类型（不要用default按钮，太重）
- 最多3个操作，超过用"更多"下拉

### 8.3 分页
- 右对齐
- 显示总条数 + 每页条数选择器 + 页码
- 默认每页20条（不是10条，不是50条）

---

## 九、表单规范

### 9.1 表单布局
- 标签在上方（不要左侧label，浪费横向空间）
- label字号：14px，`#4e5969`
- 输入框高度：32px（default size）
- 表单项间距：20px（垂直）
- 必填标记：红色 `*` 在label前

### 9.2 表单按钮
- 提交/保存：主色按钮，在右下角
- 取消：默认按钮，在提交按钮左侧
- 按钮间距：12px

## 十、交互规范

### 10.1 加载状态
- 页面初始加载：骨架屏（Skeleton）
- 按钮提交：按钮loading态
- 表格加载：表格内置loading
- 图表加载：图表内置loading

### 10.2 反馈
- 成功操作：`ElMessage.success()`，3秒后消失
- 失败操作：`ElMessage.error()`，不自动消失，需手动关闭
- 删除确认：`ElMessageBox.confirm()`，红色危险按钮
- 批量操作：显示选中数量

### 10.3 路由跳转
- 菜单点击：router.push（SPA导航）
- 表格行点击：跳转详情页
- 面包屑：可点击返回上级

- **点击子菜单项**：跳转到对应路由
- **折叠按钮**：在header左侧，折叠后只显示图标

## 十二、第三轮迭代补充

### 12.1 信息密度控制
- 运维平台属于"高信息密度"场景，但不能拥挤
- **关键技巧**：用间距和分组代替分割线
- 每个信息块不超过7个元素（Miller's Law）

### 12.2 视觉层次（Z-pattern）
- 左上：Logo/品牌
- 右上：搜索/通知/用户
- 左中：导航
- 右中+下方：核心内容
- 内容从左到右、从上到下是自然阅读流

### 12.3 告警状态在UI中的视觉权重
```
Critical → 红色背景 + 白字（最强）
High     → 红色文字 + 红色边框
Medium   → 橙色文字
Low      → 蓝色文字
Info     → 灰色文字
```
绝不允许：所有告警同一颜色显示

### 12.4 空状态设计
每个列表/表格/图表的空状态必须包含：
1. 图标（大尺寸，64px）
2. 说明文字（一句话）
3. 操作引导（按钮/链接）

禁止：只显示"No Data"或空白。
