### ⚠️ 分类问题分析

| 问题 | 现状 | 设计文档要求 | 建议 |
|------|------|-------------|------|
| **故障处置台**放在「指挥中心」目录 | `features/command-center/IncidentResponsePage.vue` | 故障处置台 = 独立工作台(§2.4) | 应迁到 `features/incident-response/`（目录已存在但为空） |
| **AI 诊断**放在「指挥中心」目录 | `features/command-center/AIDiagnosisPage.vue` | AI 是独立工作台(§2.6) | 应迁到 `features/aiops-knowledge/` |
| **AutomationPage**放在「指挥中心」 | `features/command-center/AutomationPage.vue` | 属自动化编排台 | 应迁到 `features/automation-orchestration/` 或删除（功能已被脚本/Playbook/策略/执行覆盖） |
| **工单**放在「监控事件」目录 | `features/monitoring-event/TicketPage.vue` | 工单是独立领域(§2.7) | 应迁到 `features/ticket/`（目录已存在但为空） |
| **采集器管理**放在「资产配置」 | `features/asset-config/CollectorPage.vue` | 采集器属于采集与状态中心(§5.3) | **当前位置合理**——从UX角度，用户在资产配置台操作采集器更符合直觉 |
| **KnowledgePage 重复** | `features/asset-config/KnowledgePage.vue` (113行,旧) + `features/aiops-knowledge/KnowledgePage.vue` (447行,新) | 只应有一个 | 删除 `asset-config/` 下的旧文件 |
| **侧边栏分组** | 6组(指挥中心/资产配置/监控事件/自动化/知识/管理) | 设计文档7组(§6) | 应调整为7组：拆出「故障处置」或合并「指挥中心」为仪表盘 |

### ✅ 分类合理性结论

从用户体验角度，**当前 6 组侧边栏比 7 组更合理**：
- 「故障处置」和「AI 诊断」使用频率低于日常操作，放入「指挥中心」子菜单减少侧边栏负担
- 但**代码目录应纠正**，使文件位置与逻辑归属一致
- 工单应独立目录，因为工单和告警是完全不同的交互模式

**建议最终侧边栏结构**：

```
指挥中心 → 运维指挥台 / 故障处置 / AI 诊断
资产配置 → 资产列表 / 资产发现 / 资产分组 / 凭证管理 / 配置管理 / 采集器
监控告警 → 监控总览 / 事件列表 / 告警中心 / 告警规则
工单协同 → 工单列表                      ← 从监控事件拆出
自动化   → 脚本库 / Playbook / 策略管理 / 执行历史
知识中心 → 知识库 / 知识导入              ← 更名为"知识中心"
平台管理 → 用户/角色/API Key/配置/状态/备份/审计
```
